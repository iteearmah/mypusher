# Hosting on DigitalOcean VPS (Droplet)

This guide explains how to host this custom Pusher-compatible server on a DigitalOcean Droplet using Ubuntu, Nginx, and PM2.

## 1. Create a Droplet
1. Log in to DigitalOcean and click **Create** -> **Droplets**.
2. Choose **Ubuntu** (latest LTS) as the OS.
3. Select a plan (the basic $6/month plan is usually enough for small projects).
4. Choose a data center region near your users.
5. Select **SSH Keys** for authentication (recommended).
6. Click **Create Droplet**.

## 2. Initial Server Setup
Connect to your server via SSH:
```bash
ssh root@your_droplet_ip
```
Update your system:
```bash
sudo apt update && sudo apt upgrade -y
```

## 3. Install Node.js
Install Node.js using NodeSource (for the latest LTS):
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

## 4. Deploy Your Code
1. Clone your repository:
   ```bash
   git clone https://github.com/your-username/mypusher.git /var/www/mypusher
   ```
2. Navigate to the directory:
   ```bash
   cd /var/www/mypusher
   ```
3. Install dependencies:
   ```bash
   npm install --production
   ```

## 5. Configure PM2
PM2 keeps your server running in the background.
1. Install PM2:
   ```bash
   sudo npm install -g pm2
   ```
2. Start the server:
   ```bash
   pm2 start server.js --name "mypusher"
   ```
3. Set up PM2 to start on boot:
   ```bash
   pm2 startup systemd
   # Run the command generated by the output of the above command
   pm2 save
   ```

## 6. Configure Nginx as a Reverse Proxy
Nginx will handle incoming HTTP/HTTPS traffic and forward it to your Node.js app.
1. Install Nginx:
   ```bash
   sudo apt install -y nginx
   ```
2. Create a new Nginx configuration file:
   ```bash
   sudo nano /etc/nginx/sites-available/mypusher
   ```
3. Paste the following configuration (replace `pusher.yourdomain.com` with your actual domain or subdomain):
   ```nginx
   server {
       listen 80;
       server_name pusher.yourdomain.com;

       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection "upgrade";
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```
4. Enable the configuration and restart Nginx:
   ```bash
   sudo ln -s /etc/nginx/sites-available/mypusher /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

## 7. Setup SSL (Let's Encrypt)
1. Install Certbot:
   ```bash
   sudo apt install -y certbot python3-certbot-nginx
   ```
2. Obtain an SSL certificate (replace with your actual domain or subdomain):
   ```bash
   sudo certbot --nginx -d pusher.yourdomain.com
   ```
3. Follow the prompts. Certbot will automatically update your Nginx configuration to use HTTPS.

## 8. Firewall Setup
Ensure your firewall allows HTTP, HTTPS, and SSH:
```bash
sudo ufw allow 'Nginx Full'
sudo ufw allow OpenSSH
sudo ufw enable
```

## 9. Client Configuration
Update your `index.html` or client code to point to your new domain:

```javascript
const pusher = new Pusher(PUSHER_KEY, {
  cluster: PUSHER_CLUSTER,
  wsHost: 'pusher.yourdomain.com', // Use your domain or subdomain
  wsPort: 443,
  wssPort: 443,
  forceTLS: true,
  disableStats: true,
  enabledTransports: ['ws', 'wss']
});
```
